core4os version 4.1

We introduce core4os version {{ version }} with a completely revised app management system and look and feel. Under the hood the job automation framework/API integration has been refactored and multiple minor and not so minor improvements have been implemented.

These changes have been delivered and tested throughout the last year. It has been a tremendous team effort.

NOTE that it was not possible to migrate your existing boards. Therefore we kindly ask you to create and setup new boards.

App/API Framework
  • improved job api, previous job management API is deprecated
  • new CoreStore class for cross-user storage based on user app://store
  • custom card handling and description, .card method depreciation
  • refactored /_info endpoint with pagination
  • refactored /about handler with general system information
  • Install Python package wheel by default with coco --init
  • default target for links is "blank"
  • Improved docstrings
  • changed default route from 0.0.0.0 to localhost
  • retrieve menu items, contact, profile link from configuration/store
  • retrieve theming default values from store
  • retrieve default contact email from store
  • retrieve menu from store
  • fix email message (password URL)
  • fix collections depreciation
  • reset password must not fail if email not exists
Deployment System
  • Deploy webapps with built distribution directory (./dist)
  • Build distribution directory ./dist with coco --dist
  • No default R packages, mongolite/feather are not installed by default
  • coco --cran to install R package requirements with r.txt
  • Deploy with pip package manager; core4build is deprecated
  • improved installation and testing guidelines
App/Widget Manager
  • new layout
  • Scrollbar with board list
  • improved handling of tags with default tags and search dialog
  • widget search syntax based on pql
  • support user avatar
  • SVG image for system label and icon
  • dark styling for profile page
  • default icon
Job Monitoring (Comoco)
  • refactored job handling
  • display job args to job detail view
Role Management
  • add confirmation dialog before user deletion
  • allow all characters with username except whitespace
Help Screens
  • always show qual_name, short title
